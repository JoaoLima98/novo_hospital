# Documento de Visão: Sistema de Fluxo Hospitalar

## 1. Contexto de Negócio

A proposta ao criar este sistema é substituir o atual sistema de acompanhamento hospitalar utilizada atualmente no contexto do **Hospital Regional de São Paulo do Potengi - RN**. No hospital, cada atendimento exige a criação manual de uma ficha do zero, mesmo que o paciente já tenha sido atendido anteriormente. 

A proposta é que nosso sistema agilize os atendimentos, já que o paciente vai precisar dar suas informações apenas a primeira vez, durante o registro no sistema. Se o paciente precisar voltar uma segunda vez, na qual seu registro já constará no sistema, o/a recepcionista poderá apenas consultar o paciente através do seu CPF ou CNS, e encaminhá-lo diretamente a triagem.

## 2. Escopo do Produto

- **Funcionalidades:**
    - Gerenciar Paciente
    - Acompanhar e Atualizar Registro do Paciente
    - Atualizar Status do Paciente
    - Verificar Histórico do Paciente
    -  Criar Autenticação
    -  Diagnosticar Paciente
    -  Prescrever Medicamento
    -  Gerar Relatórios
    -  Fazer Triagem
    -  Gerenciar Estoque de Medicamentos

- **Fora do Escopo:**
    - Portal do Paciente;
    - Gestão de Internação/Leitos;
    - Integração com Laboratório/Radiologia;
    - Acompanhamento/Controle Cirúrgico;
    - Fase de infusão;
 
## 3. Justificativa
- Maior velocidade no atendimento desde o primeiro cadastro.
- Dados do paciente mantidos para futuras consultas.
- Dados de atendimentos anteriores dos pacientes.
- Gerenciamento preciso do estoque de medicamentos.
